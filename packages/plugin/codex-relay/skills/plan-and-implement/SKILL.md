---
name: plan-and-implement
description: Plan a repository change first, then implement only after approval.
---

Use this workflow for Slack-triggered implementation tasks.

1. Inspect only the relevant files and tests.
2. Produce a concise plan with proposed edits, tests, and risks.
3. Wait for explicit execution approval from the harness.
4. During implementation, stay within the approved scope.
5. End with changed files and verification results.
